# 卦叽 App 图标替换方案

> 日期：2026-06-09  
> 状态：已实施（2026-06-09）  
> 关联：`app.json`、`assets/`、`src/components/GuajiLogo.tsx`

---

## 1. 背景与目标

### 现状

| 项目 | 说明 |
|------|------|
| 配置 | `app.json` 已声明 5 个图标路径 |
| 实际素材 | 全部为 Expo 脚手架默认蓝色箭头图标 |
| 品牌不一致 | App 内使用 `GuajiLogo`（镜头环 + ☯ + 小胡子），桌面图标完全不同 |
| Android 背景色 | `#E6F4FE` 浅蓝，与 App 暗色赛博风冲突 |

### 目标

1. 桌面/商店图标与 App 内 `GuajiLogo` 视觉语言统一
2. 小尺寸（48px）下仍可辨认
3. 覆盖 iOS / Android / Web 全平台
4. 后续可复用同一主稿生成分享图、通知图标等

---

## 2. 品牌元素提取

来自 `cyberTheme` 与 `GuajiLogo`：

| 元素 | 色值 / 形态 | 图标中是否保留 |
|------|-------------|----------------|
| 背景 | `#0A0A0F` | ✅ 主背景 |
| 镜头外环 | 圆环，描边 `#00F5FF` 3px 等效 | ✅ 主识别形 |
| 内环光晕 | `rgba(0,245,255,0.4)` | ✅ 细内环 |
| 八卦圆 | 填充 `rgba(0,245,255,0.1)`，描边 `#00F5FF` | ✅ |
| 太极符号 ☯ | `#00F5FF` | ✅ 核心符号 |
| 小胡子 | 两条弧线 | ❌ 48px 以下不可见，仅 Splash 保留 |
| 紫色点缀 | `#6B4C9A` | 可选：外圈微弱光晕 |

### 图标构图（推荐）

```
┌─────────────────────────┐
│    #0A0A0F 纯色背景      │
│                         │
│      ╭─────────╮        │
│     │  ╭─────╮  │       │  ← 外镜头环 #00F5FF
│     │  │  ☯  │  │       │  ← 内八卦圆 + 太极
│     │  ╰─────╯  │       │
│      ╰─────────╯        │
│                         │
└─────────────────────────┘
```

- 主图形占画布 **~72%** 直径（留足 iOS 圆角裁切边距）
- 图形垂直居中（无胡子，无需下移）
- 禁止文字、「卦叽」字样（多语言与极小尺寸不友好）

## 3. 三种实现路径

### 方案 A：设计稿导出 PNG（推荐）

设计师 / AI 生成一张 **1024×1024** 主稿，再按规格裁切各平台文件。

| 优点 | 缺点 |
|------|------|
| 质量可控，商店审核友好 | 改稿需重新导出 |
| 与 Figma/Sketch 工作流一致 | 需人工核对各尺寸 |

**适用**：首发上架、追求视觉精细度。

### 方案 B：SVG 主稿 + 脚本批量导出

用矢量描述镜头环 + 太极，Node/`sharp` 脚本导出多尺寸 PNG。

| 优点 | 缺点 |
|------|------|
| 改色/改比例可脚本化 | ☯ 符号字体渲染各平台可能不一致 |
| 版本管理友好（文本 diff） | 初版搭建成本略高 |

**适用**：图标会频繁迭代、需要 CI 自动生成。

### 方案 C：程序渲染 `GuajiLogo` 截图

用 `react-native-view-shot` 或 Playwright 渲染现有组件导出 PNG。

| 优点 | 缺点 |
|------|------|
| 与 App 内 100% 一致 | 默认含胡子，小尺寸糊；需改 props |
| 无需额外设计工具 | 发光/抗锯齿不如专业导出 |

**适用**：快速验证，不建议作为最终上架素材。

### 推荐

**方案 A** 作为上架素材，**方案 C** 仅作快速原型对照。若后续迭代频繁，再迁移到方案 B。

## 4. 资产清单与规格

### 4.1 文件映射（替换 `assets/` 下现有文件）

| 文件 | 尺寸 | 用途 | 设计要点 |
|------|------|------|----------|
| `icon.png` | **1024×1024** | iOS 通用图标源图 | 完整构图，无透明边 |
| `android-icon-foreground.png` | **1024×1024** | Android 自适应前景 | 仅镜头环+太极，**透明背景**；图形限制在中心 **66%** 安全区 |
| `android-icon-background.png` | **1024×1024** | Android 自适应背景 | 纯色 `#0A0A0F` 或微弱径向渐变（中心 `#1A1A2E` → 边缘 `#0A0A0F`） |
| `android-icon-monochrome.png` | **1024×1024** | Android 13+ 主题图标 | 单色剪影：外环 + 内圆 + 简化太极，白色 `#FFFFFF` 透明底 |
| `favicon.png` | **48×48**（或 32/48 多尺寸） | Web 标签页 | 从主稿缩小，可略增描边粗细 |

### 4.2 `app.json` 需同步修改

```json
"android": {
  "adaptiveIcon": {
    "backgroundColor": "#0A0A0F",
    "foregroundImage": "./assets/android-icon-foreground.png",
    "backgroundImage": "./assets/android-icon-background.png",
    "monochromeImage": "./assets/android-icon-monochrome.png"
  }
}
```

`backgroundColor` 从 `#E6F4FE` 改为 `#0A0A0F`，与暗色主题一致（应用启动过渡更自然）。

### 4.3 可选增强（非必须）

| 配置项 | 说明 |
|--------|------|
| `expo.splash` | 增加启动屏：背景 `#0A0A0F` + 居中 `GuajiLogo` 静帧 |
| `ios.icon` | Expo 56 可从统一 `icon.png` 生成，无需单独文件 |
| `web.favicon` | 可补充 `favicon.ico` 多尺寸（Expo export 时处理） |

## 6. 实施步骤（批准后执行）

```
1. 设计评审
   └─ 确认概念稿 → 定稿 1024 主稿

2. 导出资产
   ├─ icon.png
   ├─ android-icon-foreground.png（透明底）
   ├─ android-icon-background.png
   ├─ android-icon-monochrome.png
   └─ favicon.png

3. 更新 app.json
   └─ android.adaptiveIcon.backgroundColor → #0A0A0F

4. 本地验证
   ├─ npx expo start → 开发菜单查看图标
   ├─ npx expo prebuild（若已有 native 目录需重建）
   └─ 真机 / 模拟器检查 48px、180px 清晰度

5. 上架前检查
   ├─ iOS：无 alpha 通道（icon.png 需不透明底）
   ├─ Android：前景安全区无裁切
   └─ Web：favicon 在标签页可辨认
```

## 7. 验收标准

- [x] 主屏幕图标可一眼联想到 App 内 Splash 的 `GuajiLogo`
- [x] 48×48 像素下太极与圆环仍可区分
- [x] Android 自适应图标在各种 launcher 形状（圆/方/圆角方）下无关键元素被裁
- [x] 浅色/深色系统主题下 Android monochrome 图标正常
- [x] 无 Expo 默认蓝色箭头残留

---

## 8. 风险与规避

| 风险 | 规避 |
|------|------|
| ☯ 字符在不同字体下形态不一 | 图标内用矢量绘制阴阳鱼，不用文字符号 |
| 青色 `#00F5FF` 在白色背景上对比不足 | 仅用于深色底；Android 前景为透明底叠深色背景 |
| 已有安装用户图标缓存 | 版本号 bump + 重新安装/EAS build 后生效 |
| `assets/tarot/` 等大体积资源与图标无关 | 本次只替换根级 5 个图标文件，不动 tarot 资源 |
